# Easy-tactics - HEMA Fighter Knowledge Agent — Дизайн-документ

## 1. Обзор

**Название:** Easy-tactics - HEMA Fighter Knowledge Agent  
**Назначение:** AI-агент для анализа техники фехтовальщиков — поиск на хемагоне, парсинг интернета, анализ видео, формирование саммари и рекомендаций  
**Целевая аудитория:** Тренеры и спортсмены в фехтовании  
**Стартовая точка:** CLI/API + Telegram бот

## 4. Workflow (MVP)

### Этап 1: Поиск бойца

```
Пользователь → вводит имя бойца
    ↓
Поиск в базе
    ├── Есть → показать профиль + дата обновления
    │   └── опции: обновить / дополнить / посмотреть
    └── Нет → поиск на хемагоне
        ↓
Варианты профилей → пользователь выбирает
    ↓
Создание профиля в базе
```

### Этап 2: Поиск турниров

```
Парсинг хемагона → найдены турниры (X штук)
    ↓
Рекомендации: "интересные турниры: Y (недавно, сильные соперники)"
    ↓
Пользователь выбирает: все / рекомендованные / конкретные
```

### Этап 3: Поиск боев

```
Сетка боев в выбранных турнирах
    ↓
Парсинг → найдены бои (Z штук)
    ↓
Рекомендации: "интересные бои: ..."
    ↓
Пользователь выбирает бои для анализа
```

### Этап 4: Анализ

```
Для каждого выбранного боя:
    1. Поиск видео (VK/YouTube)
    2. Если видео найдено → AI анализ
    3. Если видео не найдено → пометить "видео не найдено"
    ↓
Агрегация → общее саммари
    ↓
Вывод пользователю
```

### Этап 5: Дополнение/редактирование

```
Пользователь может:
├── Ручное редактирование markdown
├── Добавить наблюдение (текст)
│   └── AI интегрирует в саммари
├── Добавить видео (файл + комментарий)
│   └── AI анализирует комментарий
├── Добавить теги
├── Отметить как "неверное"
```

### Этап 6: Загрузка своих видео (отдельный воркфлоу)

```
Пользователь → загружает видео + комментарий
    ↓
Выбор: новый боец / существующий боец
    ↓
AI анализирует комментарий → добавляет в наблюдения
    ↓
Предложение: разобрать видео (AI vision)
```

## 5. MCP серверы

### 5.1. Hemagon MCP

**Функции:**
- Поиск бойца по имени
- Парсинг профиля (фото, статистика, турниры)
- Парсинг сетки боев турнира
- Парсинг списка турниров бойца

**Ограничения:**
- Защита от парсинга → возможно Puppeteer/Playwright

### 5.2. VK MCP

**Функции:**
- Поиск сообществ турниров
- Поиск видео в группах
- Получение метаданных видео

**Ограничения:**
- Закрытые группы
- Требуется авторизация

### 5.3. YouTube MCP

**Функции:**
- Поиск видео по запросу
- Получение метаданных

### 5.4. Video Analysis MCP (опционально)

**Функции:**
- AI vision для анализа видео
- Описание техники

**Сложность:**
- Дорого
- MVP — только анализ комментариев пользователя

## 6. AI/LLM интеграция

### MVP: Simple LLM

- Модель: любой API (GPT-4o, Claude, YandexGPT) + опционально локальная (Llama, Qwen)
- Использование: генерация текста по промпту, не agentic
- Контроль: чекпоинты на каждом этапе, пользователь подтверждает

### Phase 2: Agentic AI

- Может сам пройти весь воркфлоу
- Принимает решения на каждом этапе
- Требует больших затрат

### Системные промпты (скиллы)

```
skills/
├── analyze-fighter.md      # анализ техники бойца
├── generate-summary.md     # генерация саммари
├── integrate-observation.md # интеграция наблюдений
├── search-videos.md        # поиск видео
└── recommend-fights.md      # рекомендации боев для анализа
```

## 7. Версионирование

- Каждое изменение саммари = коммит в `.history/`
- Хранится: полная копия + diff + метаданные (автор, причина)
- Возможность откатить (не MVP, но заложить структуру)

## 9. Технологический стек

| Компонент | Технология |
|-----------|------------|
| API Service | Go (Fiber/Echo) |
| AI задачи | Python (опционально) |

## 10. Roadmap

## 11. Узкие места и риски

| Проблема | Решение |
|----------|---------|
| Hemagon защищён от парсинга | Puppeteer/Playwright или API |
| Видео в VK закрыты | Пользователь предоставляет доступ |
| AI vision дорого | Только для Phase 3, опционально |
| Уникальный ID бойца | Slug (Фамилия-Имя-Город) |
